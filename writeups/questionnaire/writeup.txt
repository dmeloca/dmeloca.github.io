==========================
   Questionaire Writeup
==========================

So for this challenge we get test.c and an test that is executable. I other hand we have
the endpoint that HTB give to us.

if we connect ourselfs to this ip via netcat or other software, we are asked to answer 
certain questions.

0x1. Is this a '32-bit' or '64-bit' ELF?
	- To answer this we use the `file` (is recommended to check the man page of this
	command).
```bash
test: ELF 64-bit LSB executable, x86-64, version 1 (SYSV), dynamically linked, interpreter /li
b64/ld-linux-x86-64.so.2, BuildID[sha1]=5a83587fbda6ad7b1aeee2d59f027a882bf2a429, for GNU/Linu
x 3.2.0, not stripped 
```
	- As you can see this command reveal the answer.

0x2. What's the linking of the binary? 
	- Again the file command help us to answer the question.
	- Answer: `dynamic`

0x3. Is the binary 'stripped' or 'not stripped'?
	- Answer: `not stripped` -> Again by using the output of `file`

0x4. Which protections are enabled (Canary, NX, PIE, Fortify)?
	- Here we introduce another binary named `checksec`. This command could be installed
	via pwntools python library and reveal to us more information about the binary to be
	tested.
```bash
*] '/HTB/challenges/pwn/pwn_questionnaire/test'                             
    Arch:       amd64-64-little                                                               
    RELRO:      Partial RELRO                                                                 
    Stack:      No canary found                                                               
    NX:         NX enabled                                                                    
    PIE:        No PIE (0x400000)                                                             
    SHSTK:      Enabled                                                                       
    IBT:        Enabled                                                                       
    Stripped:   No 
```
	- The `checksec` command analyze the binary finding some of the known binary 
	protections (see the other post related to binary protections to know more)
	in this case, we have that the answer is `NX` because is the only protection 
	activated.
0x5. What is the name of the custom function the gets called inside `main()`? (e.g. vulnerable_function())
	- Furtunely the challenge gave us the original c code, so is more easy to understand
	the logic.
	- So, for solving this question make a `cat test.c` and read code.
```c
void main(){                                                                                  
        vuln();                                                                               
}   
```
       
0x6. What is the size of the 'buffer' (in hex or decimal)?
	- Again, just by reading the code we can answer the question.
	- In vuln, we have an array started with zeros 
	```c
	char buffer[0x20] = {0};
	```
	- Answer: `0x20` 

0x7. Which custom function is never called? (e.g. vuln())
	- Reading the logic in the code we see that the `gg()` func is never called normally.

0x8. What is the name of the standard function that could trigger a Buffer Overflow? (e.g. fprintf())
	- I suggest making a `man fgets`.
	- As you now understand how `fgets` function works, you now understand why the 
	function is vulnerable
	- Surprise, the buffer is `0x20` in size, but `fgets` reads `0x100`.
	- As you can see here exists a BOF. Later explained, patience.
	- Answer: `fgets()`

0x9. After how many bytes a Segmentation Fault occurs (in hex or decimal)?
	- As we are suggested we can make `printf 'A%.0s' {1..n} | ./test` to send n A's to 
	the input of the binary. So we start to vary the number to check when the program
	crashes.
	- Answer: 40

0xa. What is the address of 'gg()' in hex? (e.g. 0x401337)
	- Now we unlocked a powerfull tool `gdb`, this tool will be with us around our road
	to learn pwn. So we must learn, how to used it. (please check the gdb dojo at 
	pwn.college).
	- To answer this question we run `gdb test`, in *gdb* we use the `info functions`
	command, to get the functions that are on the binary.
	- The direction that appears on the first column of the output is in 64-bit, so as
	the question, ask to us to respond on 32-bit we just simply remove the zeros.

Finally, we solved the challenge.
